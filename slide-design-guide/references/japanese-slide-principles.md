# 日本式スライドデザイン原則ダイジェスト

日本のスライドデザイン専門サイト10件から抽出した、生成時・レビュー時共通の基準。
（2026-06-12 リサーチ。出典は末尾の10サイト一覧）

## 1. メッセージ設計

- **1スライド1メッセージ**。詰め込むくらいならスライドを分ける（フォントを縮めて収めるのは禁止）。
- タイトルは**体言止め or アクションタイトル**（「売上推移」ではなく「売上は3年で2.4倍に拡大」）。結論を先に言う。
- **KISSの法則**: 文は短く。スライドは「読ませる」ものではなく「見せる」もの。
- 発表で話すこととスライドに書くことを分離する（スライドは骨子、詳細は口頭/配布資料）。

出典: 森重湧太「見やすいプレゼン資料の作り方」、シリョサク、Document Studio

## 2. タイポグラフィ

- 和文は**太めのゴシック体**: Noto Sans JP / ヒラギノ角ゴ / 游ゴシック / メイリオ。明朝は和風・高級感の見出し演出のみ。
- **本文18pt相当以上**（会場投影なら24pt以上）。キャプション最小でも14pt相当。
- **ジャンプ率**（タイトル:本文のサイズ比）は1.5〜2.5倍を確保してメリハリを出す。
- 行間は**1.3〜1.5**（和文は欧文より広めに）。長い行は40字以内で折る。
- 太字の多用禁止。強調は「サイズ・ウェイト・色」のうち1〜2手段に統一。
- 英数字は欧文フォント（和文フォントの英数字は使わない）が望ましい。
- **実装の要点（必ず適用）**: `font-feature-settings:"palt" 1`（詰め組）/ `line-break: strict`（禁則）/
  見出しに `text-wrap: balance`（1文字孤立回避）/ 和欧混植フォントスタック。
  → コピペ基盤CSSと事故対処表は `references/japanese-typography.md`。

出典: 伝わるデザイン、The Power of PowerPoint 基礎編

## 3. 配色

- **ベース70% : メイン25% : アクセント5%** の面積比。
- 使う色は**無彩色（白/黒/グレー）＋3色以内**。迷ったら1色＋グレー。
- **原色・高彩度を避ける**。彩度を一段落とすと日本のビジネス資料らしい落ち着きになる。
- 色だけで意味を伝えない（形・ラベル併用）。コントラスト比は**WCAG AA（4.5:1）以上**。
- 配慮が必要な場面は**CUD推奨配色セット**（アクセント9色+ベース7色+無彩色4色）から選ぶ。
- **和風テイスト**は日本の伝統色から: NIPPON COLORS / 和色大辞典でhexを取得
  （例: 藍 #165E83、朱 #EB6101、鶯 #928C36、生成り #FBFAF5）。伝統色＋明朝見出し＋余白多めが定石。
- **テイスト別の伝統色セット（hex付き）は `references/wairo-palette.md`**。配色はそこから1セット選び、
  ベース70:メイン25:アクセント5で固定。ベースは純白より生成り #FBFAF5。

出典: The Power of PowerPoint、Document Studio、CUDO、NIPPON COLORS / 和色大辞典

## 4. レイアウト

- **デザイン4原則**: 整列（揃える）・近接（関係あるものは近く）・反復（同じ役割は同じ見た目）・対比（重要度の差を見た目の差に）。
- **視線の流れ**: 横書きはZ型/F型。重要情報は左上→右下の動線上に置く。
- **ピクセル整列**: ガイド/グリッドで端を揃える。中途半端な1〜2pxのズレが「素人感」の最大要因。
- **余白を恐れない**: 余白はデザインの一部。端まで詰めない（スライド外周に最低60px/1600px幅相当のマージン）。
- **縦軸を設計する**: 固定キャンバスでは flex中央寄せ / space-between で下部の死に余白を作らない。
- 同じレイアウトを2枚以上連続させない（タイトル/2カラム/データ/引用/ベント等を使い分ける）。

### 4.5 「間（ま）」を効かせる ― 死に余白との決定的な違い

日本のデザインは**間（負の空間）を内容として扱う**。ただし「良い間」と「事故の死に余白」は別物。
生成・レビュー両方でこの判定を当てる:

- **良い間** = 強い焦点が1つある＋その周りを**端を揃えたまま大胆に空ける**（非対称でよい）。
- **死に余白** = 焦点が無いまま中途半端に空いている／中央均等配置の下にポッカリ空く／グリッドに乗っていない。
- ルール化:
  - 1スライドに**主役を1つだけ**大きく置く（指標・見出し・図のどれか）。残りは思い切って空ける。
  - 空きは必ず**グリッド/ガイドに乗せて端を揃える**。乗っていない1〜2pxのズレ＋空きが最大の素人感。
  - アンカーは中央均等寄せより**左上 or 右下**に寄せると緊張感が出る（視線の動線 Z/F に沿う）。
  - 「下に200px空いた」を見たら、(a)主役を拡大 (b)space-betweenで上下に張る (c)余白を残すなら端を揃えて意図的に、のどれかにする。
- 和テイストは特に**余白多め＋明朝見出し＋伝統色**の三点で「静けさ／品」を作る。

出典: Document Studio（レイアウト4原則）、good presen!（視線誘導）、伝わるデザイン

## 5. 図解・チャート

- **箇条書きで3階層を超えたら図解化**（フロー/対比/マトリクス/ピラミッド等の「型」に当てる）。
- 矢印・線・角丸・影のスタイルはデッキ全体で統一する。
- グラフは1スライド1つ。伝えたい1点だけ色を付け、他はグレー（フォーカスの原則）。
- 表は罫線を減らし、ゼブラ縞 or 余白で行を区切る（html2pptxでは div+flexbox で組む）。

出典: シリョサク（図解の型）、伝わるデザイン

## 6. 日本のビジネス文脈

- 決算説明・IR系は「結論サマリ→根拠データ→補足」の構造化と、ページ番号・出典表記・凡例の丁寧さが信頼感を作る（SlideLandの決算/統合報告書タグが実例庫）。
- 社外提案は表紙に日付・宛先・社名。フッターにブランド名＋ページ番号を置くと「ちゃんとした資料」に見える。
- 過剰な英語キャッチコピーより、短い日本語の言い切りが伝わる。

## 7. セルフレビュー チェックリスト

生成後にスクリーンショットを撮り、以下を確認して最低1周改善する:

- [ ] 各スライドのメッセージを一文で言えるか / タイトルが結論になっているか
- [ ] 本文18pt相当未満の文字がないか / ジャンプ率1.5倍以上あるか
- [ ] 無彩色+3色以内か / 原色がないか / コントラストAAを満たすか
- [ ] 端の整列が揃っているか / 下部に死に余白がないか
- [ ] 同一レイアウトが連続していないか
- [ ] 箇条書きの図解化余地がないか / 矢印・装飾が統一されているか
- [ ] （和風なら）伝統色・明朝・余白の三点が揃っているか
- [ ] **実在情報の混入がないか**（実在の社名・サービス・人名・ロゴ・登録商標/規格/イニシアチブ・住所/〒/電話・実数値・逐語コピー）。
      固有名は架空に統一し、第三者商標は汎用語に置換。外部画像/実URL/実メール無し（Google Fonts と w3.org のみ可）。詳細は SKILL.md セクション4。
